# Ops Bot (Phase 1)

This bot provides lightweight operational automation without making production code changes.

## What it does

- Runs health checks on a schedule.
- Opens incident issues when endpoints fail.
- Creates one daily digest issue summarizing open ops work.

## What it does not do

- No direct commits to `main`.
- No auto-merge.
- No auto-deploy actions.

## Required GitHub secrets

- `OPS_HEALTH_ENDPOINTS`: comma-separated full URLs to probe (for example: `https://api.example.com/health,https://app.example.com`).

## Workflows

- `.github/workflows/ops-health-check.yml`
- `.github/workflows/ops-daily-digest.yml`

## Labels to create

- `ops:incident`
- `ops:degraded`
- `ops:autogenerated`
- `source:in-app-report`
- `source:health-check`

## Local run

```bash
OPS_HEALTH_ENDPOINTS="http://localhost:4000/health" node scripts/ops/health-check.mjs
node scripts/ops/daily-digest.mjs
```

For local digest testing, set `GITHUB_TOKEN` and `GITHUB_REPOSITORY` in your shell.
