## Status Key

- `spec` = requirements and acceptance criteria defined
- `design` = UX/technical design in progress
- `build` = implementation in progress
- `qa` = testing and verification in progress
- `done` = released and validated

## How To Use This Board

1. Pick one row and create an issue from the `Roadmap Feature` issue template.
2. Paste the created issue link into the `GitHub Issue` column.
3. Fill `Owner`, `Sprint`, and `Readiness`.
4. Update checkbox to `[x]` only after release validation is complete.

## Weekly Review Rhythm

- Review all `A (Now)` items first.
- Promote `B (Next)` items only when a `Now` slot is cleared.
- Keep `Deferred` items visible but unplanned until strategy changes.
